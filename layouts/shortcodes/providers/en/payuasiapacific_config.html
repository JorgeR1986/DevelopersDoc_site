ConfigurationRequired/Optional
In the PaymentsOS Control Center, configure the following credentials:
  • clientId: The client ID provided by PayU Asia Pacific.
  • clientSecret: The client secret provided by PayU Asia Pacific.
  • merchantId: The merchant ID provided by PayU Asia Pacific.
  • accountId: The ID represents the identifier of your account with PayU Asia Pacific. If you do not configure this field for non-card payment methods, then you must pass the accountId in the Create Authorize request. Note: For card payment, if you do not configure this field or pass the accountId in the Create Authorize request, then PayU Asia Pacific will use the BIN number to route the payment to the most optimal payment provider.
